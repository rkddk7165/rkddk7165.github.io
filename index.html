01

Hot 게시글 조회수 병목

동일 postId에 트래픽이 몰릴 때 요청마다 DB write가 발생하던 구조를 Caffeine 기반 Write-Behind로 전환했습니다.

2.27s → 0.45~0.59s p95 지연 약 74~80% 개선

검색 API 개선

200ms → 11~30ms OpenSearch 기준 후보 탐색 약 78~92% 감소
03

목록 조회 N+1 제거

댓글 수 집계를 게시글마다 호출하던 구조를 2-Phase 조회로 바꿔 API 1회 호출당 쿼리 수를 상수화했습니다.

23 → 2 queries 쿼리 수 약 91% 감소
04

댓글 COUNT 인덱스

0.642ms → 0.093ms 단일 쿼리 약 85.5% 개선
